## 1. 背景与目标 (Why)

### 1.1 业务背景

你的核心业务是**包车带司机**的入境游定制服务。行程天数直接决定包车天数，酒店数量直接决定房费，景点直接决定门票——**行程和报价本质是一件事的两面**。你在整条链路中处于**枢纽位置**，一对多对接上下游：

```
多个境外旅行社          一手地接社（PandaKing）        多个省地接社
   ┌──────────┐         ┌──────────────┐         ┌──────────┐
   │泰国A旅行社│ ←─报价── │              │  ──成本询价──→ │新疆A旅行社│
   │旅行社B   │ ←─报价── │   总集成      │  ──成本询价──→ │四川社    │
   │旅行社C   │ ←─报价── │   加利润      │  ──成本询价──→ │云南社    │
   └────┬─────┘         └──────────────┘         └──────────┘
        │ 加价                    ↑                      ↑
        ↓ 报游客              加利润报旅行社           协助规划定制行程+成本价
     境外游客
```

> **命名约定** ★（v0.9.1 新增）：
> - **境外旅行社**（示例名）→ **泰国A旅行社**（其他旅行社可命名为"旅行社B""旅行社C"等）
> - **一手地接社（我方）**（示例名）→ **PandaKing**
> - **省地接社**（示例名）→ **新疆A旅行社**（其他省地接社可命名为"四川社""云南社"等）
> - **泰国A旅行社和新疆A旅行社在系统信息流转中是被隔离的，互相不可见**——境外旅行社和省地接社之间不直接对接，所有信息经一手地接社（PandaKing）中转
> - 角色定义表中的角色名称保持不变（"境外旅行社销售""一手地接社运营""省地接社运营"等），具体示例名称仅用于场景描述和流程图

**四方角色定义** ★（v0.9 更清晰定义）：

| 角色 | 职责 | 数量 | 关系 |
|---|---|---|---|
| **境外游客** | 提需求（目的地/天数/预算/偏好），反馈修改意见 | 多人 | 不直接对接，需求经旅行社转达 |
| **境外旅行社路线规划人员** | 对接游客、整理需求、转达给一手地接社、**在系统内参与行程+报价反馈**、在旅行社报价基础上**加利润生成对游客报价** | 多家×多人/家 | 上游客户，你给他们报价 |
| **中国一手地接社（我方）** | 总集成：路线规划、成本询价、加利润报价、资源预订；**常规路线自行规划+报价**，定制路线请省地接社协助；**包车带司机是核心业务** | 1家 | 你是枢纽 |
| **各省地接社（省地接社）** | 协助规划当地行程、提供成本报价（酒店/包车/门票/服务费）；**v0.9 新增：在 H5 内接收成本询价、填写成本价、保存并通知** | 多家 | 下游供应商 |

**关键数量关系**：
- 1个 PandaKing（一手地接社） ↔ N个境外旅行社（如泰国A旅行社等） ↔ M个境外游客（每个旅行社带多个游客）
- 1个 PandaKing ↔ K个省地接社（按目的地分配，如新疆A旅行社、四川社、云南社等）
- 同一时间段内在跟进的行程可能有 **5-15条**，每条都可能在不同版本阶段

> **四个关键认知** ★（v0.9.1 更新）：
> 1. 一手地接社与省地接社是**协作关系**。**常规/成熟行程**可由一手地接社自行规划和报价，提高效率；**定制/新目的地行程**由省地接社提供行程规划和成本报价，一手地接社加利润后对外报价。**推荐"自行优先，按需转协作"**：先自行出初版抢占客户，需要时再引入省地接社细化。★（v0.9 新增：省地接社可通过 H5 接收成本询价并填写成本价，不再仅靠微信口头沟通）★（v0.9.1 新增：省地接社不仅填写成本价，还可在 H5 内编辑分配给自己的行程内容、反馈行程规划修改建议，成为行程规划的积极参与者）
> 2. **微信仍是主沟通通道**，系统不替代微信，而是给微信装上"结构化引擎"——每个客户、每次行程修改、每份报价都有专属 H5 链接，带标注粘贴到微信，让微信群里的沟通有归属、有版本、可回溯。★（v0.9 强化：行程报价合并后"保存并通知"一条 H5 链接含行程+报价；省地接社侧也通过 H5 闭环完成成本询价）★（v0.9.1 新增：境外旅行社销售可填写初步行程需求并提交给一手地接社，作为行程规划输入参考）
> 3. **沟通分双通道**：简单事项在微信直接聊（不强制进系统，可选手动备注留痕）；复杂事项（行程修改、报价确认、需求变更、成本询价、行程规划反馈）通过 H5 系统交互确认并记录沉淀，有价值的交互沉淀为知识库。
> 4. ★（v0.9 新增）**行程和报价是一件事的两面**：包车带司机是核心业务，行程天数决定包车天数，酒店数量决定房费，景点决定门票。行程修改必然影响报价，因此 v0.9 将行程和报价合并为「行程报价单」单一模块，修改联动不遗漏。
> 5. ★（v0.9.1 新增）**账号管理简化为邀请制**：管理者邀请→员工微信授权→直接获得权限，取消审核流程，首次打开H5完成微信授权即获得权限，降低使用门槛。
> 6. ★（v0.9.1 新增）**境外旅行社和省地接社信息隔绝**：泰国A旅行社（境外旅行社）和新疆A旅行社（省地接社）在系统信息流转中被隔离，互相不可见。所有信息经一手地接社（PandaKing）中转，旅行社不可指定省地接社，省地接社由一手地接社分发后关联。

### 1.2 现状问题

前期所有沟通都在**微信群**内完成，痛点集中在三个层面：

**层面一：信息归属与版本管理**

| 痛点 | 表现 | 业务影响 |
|---|---|---|
| P1 多游客多路线交叉混乱 | 5-15条路线同时在跟进，A游客的反馈回给了B旅行社，C旅行社的报价发给了D游客 | 商业隐私泄露、客户流失 |
| P2 版本管理失控 | v1/v2/v3在微信里来回发，旅行社说"按上次发的"，你不知道是哪个"上次" | 用旧版报价→亏利润；用新版但旅行社看旧版→客户混乱 |
| P3 换人交接断档 ★（v0.9 补充） | 负责某条路线的员工离职/休假，接手的人面对一堆微信聊天记录，完全不知道上下文 | 交接成本极高，可能丢单 |
| P7 历史版本丢失 | 微信聊天记录被清理或换手机，之前的修改过程完全消失 | 无法追溯、扯皮无据 |

**层面二：协作信息流转效率**

| 痛点 | 表现 | 业务影响 |
|---|---|---|
| 沟通记录无沉淀 | 每次修改只在微信里说，没有结构化记录 | 换人接手就断档（P3） |
| 重复问题反复答 | 游客常问的问题每次重新打字 | 回复慢，游客体验差 |
| 价格口径混乱 | 成本价、对旅行社报价、对游客报价混在一起说 | 报错价、泄底价（P4） |
| P5 变更后成本难核算 ★（v0.9 补充） | 改了3次酒店、换了车型、加了体验项目，最后利润到底多少全靠脑子算；**行程和报价分属两个模块，修改行程不会自动联动报价** | 利润被蚕食、报价可能亏本 |
| P9 反馈口语化难结构化 ★（v0.9 补充） | 游客说"那里不好不要了加个别的"，你理解偏差；**行程和报价的反馈是分开的，游客反馈往往同时涉及行程和价格（"换酒店+多少钱"），分开反馈割裂** | 改错方向、浪费来回 |
| 行程反馈来回低效 | 旅行社反馈修改意见靠微信口头描述，一手地接社改完再口头通知，来回多轮说不清 | 沟通效率低，容易遗漏 |
| P11 微信通知易遗漏 ★（v0.9 补充） | 同时跟进多条路线，某条的修改通知淹没在消息列表里；**多条路线同时跟进时缺乏优先级排序** | 遗漏修改、客户等急 |

**层面三：多账号协作混乱**

| 痛点 | 表现 | 业务影响 |
|---|---|---|
| 多旅行社交叉 | 旅行社 A 的客户信息发给了旅行社 B | 商业隐私泄露 |
| P6 多省地接社报价交叉 ★（v0.9 补充） | 四川社和云南社的成本报价在同一个微信聊天里，互相看到了对方的价格；**省地接社侧没有 H5 协作闭环，成本询价仍走微信** | 供应商比价、被动 |
| 分配不明确 | 不清楚某个客户行程该找哪个省地接社 | 踢皮球，延误 |
| 旅行社加价不透明 | 旅行社在系统外加价，改了多少、报给游客多少一手地接社不知道 | 无法掌握最终成交价，难以定价策略调整 |

> **v0.9 痛点覆盖对照** ★：
> 
> | 痛点 | v0.8 现状 | v0.9 解决方案 |
> |---|---|---|
> | P5 变更后成本难核算 | 行程在M2，报价在M3，修改行程不会自动联动报价 | **合并为行程报价单，行程变更时报价同步标记"需更新"** ★ |
> | P6 多省地接社报价交叉 | 有数据隔离，但省地接社侧没有H5闭环 | **扩展H5闭环到省地接社侧，成本询价走H5结构化** ★ |
> | P7 历史版本丢失/多路线无全局视图 | 有版本管理，但没有路线维度全局看板 | **新增多路线管理看板，一屏掌握所有路线状态** ★ |
> | P9 反馈口语化/行程报价反馈割裂 | H5反馈只覆盖行程修改意见 | **H5反馈区同时覆盖行程+报价** ★ |
> | P11 多路线通知遗漏 | 有超时提醒，但缺乏优先级排序 | **看板按紧急度排序，超时标红** ★ |

### 1.3 产品目标

把"微信群里的口头协作"变成"微信 + 结构化引擎"的协作模式：
- **微信仍是主沟通通道**，但每个客户、每次修改、每份报价都有**专属 H5 链接**，带标注粘贴到微信，一眼识别归属；
- **行程报价合并为单一页面** ★（v0.9）：行程区块（按天编排）和报价区块（按项按车型分列）在同一页面，行程修改后报价区块自动标记"需更新"，报价行跟随行程联动（如 Day3 换酒店→房费行自动标红待改），不再分两个模块手动同步；
- **行程报价"保存并通知"** ★（v0.9）：保存行程报价单后一键生成**一条 H5 链接含行程+报价**，复制粘贴到微信通知旅行社，旅行社在微信内打开 H5 即可查看行程+报价并反馈修改建议；
- **省地接社 H5 协作闭环** ★（v0.9）：一手地接社向省地接社发"成本询价 H5 链接"，省地接社在微信内打开 H5 查看行程需求+填写成本价→保存并通知→一手地接社收到，成本询价不再散落微信；
- **多路线管理看板** ★（v0.9）：工作台首页升级为多路线管理看板，一屏看到所有在跟进的路线/客户/当前版本/最新状态/待办，支持按状态筛选、按紧急度排序，超时标红，作为 MVP 核心功能；
- **微信驱动三方协作闭环** ★（v0.9）：闭环A（你↔旅行社：行程+报价协作）+ 闭环B（你↔省地接社：成本询价协作），全程微信驱动；
- **H5反馈同时覆盖行程和报价** ★（v0.9）：反馈可关联到具体行程天，也可关联到具体报价行，如"D3酒店太贵，建议换¥800以内的"——同时标注行程 Day3 和报价酒店行；
- **双通道沟通**：简单事项微信直接聊，复杂事项 H5 系统交互确认并记录沉淀；
- 每一次修改都有版本 + 备注 + 修改人记录；**行程调整后系统自动生成建议备注，供用户勾选确认**；
- 每一次报价都有快照，且**成本价 / 对旅行社报价 / 对游客报价分层隔离**；
- **5 级报价链路全程系统内闭环**：成本价 → 一手加利润 → 旅行社报价 → 旅行社加价 → 游客报价；
- **包车分车型报价**：同一行程按 7座/9座/14座等不同车型生成不同报价方案；
- **常规行程可自行规划报价**，不强制依赖省地接社，提高出单效率；
- 常见问题形成知识库，**系统自动提示历史答案**，帮助快速回复游客；
- **多个境外旅行社、多个省地接社账号体系**，数据严格隔离，**账号邀请权限分层下放**；
- **行程/报价 PDF 支持多语言导出**，**行程内容默认中文填写，AI 翻译引擎自动翻译目标语言**。

> - 旅行社加价方式：按百分比加成 或 按固定金额加价（二选一，**默认按固定金额** ★v0.9.1）
> - 旅行社加价后生成"对游客报价快照"（含游客报价，不含一手地接社成本价/利润）
> - 一手地接社可看到旅行社的加价金额和最终游客报价（掌握成交价，便于定价策略）
> - 旅行社在微信内打开 H5 链接即可参与行程+报价协作反馈，无需登录 PC 端
> - ★（v0.9 新增）旅行社在 H5 内的反馈可同时关联行程天和报价行，一条反馈覆盖行程+报价

> **省地接社角色能力** ★（v0.9.1 更新）：
> - 省地接社运营在微信内打开"成本询价 H5 链接"→查看行程需求+需要填写的成本项→填写成本价（含分车型）→点击"保存并通知"→一手地接社收到成本报价通知

> - 省地接社只看到 ① 自己的成本价，看不到对旅行社报价、一手利润、旅行社加价、游客报价
> - 省地接社可查看分配给自己的行程需求（v0.9.1 更新：从只读升级为可编辑，可以修改行程内容）

>   - 省地接社只能看到：分配给自己的行程需求 + 自己填写的成本价 + 自己的行程编辑和反馈

### 4.2 行程报价单（合并版）(M2) ★（v0.9 合并）

#### 4.2.1 设计目标

- **一屏掌握**：行程区块（按天编排）+ 报价区块（按项按车型分列）在同一页面

#### 4.2.2 行程规划双模式

创建客户行程报价单时，一手地接社运营选择行程模式：

| 模式 | 适用场景 | 流程 | 成本价填写方 |
|---|---|---|---|
| **自行模式** | 常规/成熟线路 | 选模板 → 自行修改行程 → 自行填写成本价 → 加利润 → 生成报价 | 一手地接社 |
| **协作模式** ★（v0.9 更新） | 定制/新目的地线路 | 选模板/创建空白 → 分配省地接社 → **发送成本询价H5链接** ★ → 省地接社在H5内填写成本价 → **省地接社"保存并通知"通知一手地接社** ★ → 一手加利润 → 生成报价 | 省地接社（通过H5） |

> **"自行优先，按需转协作"主流程**：
> - 推荐路径：一手地接社先用自行模式快速出初版行程+报价 → 后续如需当地资源时切换为协作模式
> - 切换无限制：自行 ↔ 协作可随时互转
> - 切换时自动记录到沟通时间线
> - 切换不影响已有报价快照
> - 自行模式成本价参考机制：填写成本价时系统自动展示同目的地/同类资源的历史报价参考
> - ★（v0.9 新增）协作模式下，一手地接社在行程报价单页面点击"发送成本询价"→生成成本询价H5链接→复制到剪贴板→粘贴到微信发给省地接社（详见 4.4）

#### 4.2.4 行程修改→报价联动机制 ★（v0.9 新增）

**这是 v0.9 合并版的核心机制**——行程修改后报价区块自动联动标记，防止"改了行程忘记改报价"。

**联动规则**：

| 行程变更类型 | 联动的报价行 | 联动动作 |
|---|---|---|
| 修改某天酒店 | 对应天的酒店行 | 标记"需更新"（标红） |
| 新增一天 | 所有按天计费行（酒店/包车/门票等） | 标记"需更新"+提示"新增Day N，请补充报价" |
| 删除一天 | 所有按天计费行 | 标记"需更新"+提示"已删除Day N，请核减报价" |
| 修改某天景点 | 门票行 | 标记"需更新" |
| 修改某天用餐 | 餐饮行（如有） | 标记"需更新" |
| 修改交通方式 | 包车行 | 标记"需更新" |
| 修改出行人数 | 车型方案行 | 标记"需更新"+提示"出行人数变更，请核实车型推荐" |

**联动展示**：

```
┌──────────────────────────────────────────────────────────┐
│  💰 报价区块                              [⚠ 2项需更新] 🔴│
│                                                          │
│  项目       │ 7座方案  │ 9座方案  │ 14座方案 │ 备注     │
│  ├─────────────────────────────────────────────────────┤│
│  │酒店(5晚)  │ ¥6,000  │ ¥6,000  │ ¥6,000  │五星级    │
│  │包车(5天)  │ ¥4,000  │ ¥6,000  │ ¥9,000  │含司机    │
│  │⚠ 门票     │ ¥3,000  │ ¥3,000  │ ¥3,000  │Day5景点变│
│  │⚠ 服务费   │ ¥2,000  │ ¥2,000  │ ¥2,000  │天数变更  │
│  └─────────────────────────────────────────────────────┘│
│  ↑ 标红的行需要您更新报价，行程已变更但报价未同步          │
└──────────────────────────────────────────────────────────┘
```

**操作流程**：
1. 用户在行程区块修改某天内容（如 Day3 换酒店）
2. 系统自动检测变更类型，将关联的报价行标记为"需更新"（标红高亮）
3. 报价区块顶部显示"⚠ N项需更新"汇总提示
4. 用户点击标红的报价行，更新价格
5. 更新后该行标记变为"已更新"（标绿，后恢复正常）
6. 所有"需更新"项处理完毕后，汇总提示消失
7. 如用户在仍有"需更新"项时点击"保存并通知"，系统弹出确认："仍有2项报价未更新（门票、服务费），确认保存并通知？"——用户可选择"继续保存"或"先去更新"

> **设计目标**：减少因行程变更导致报价遗漏的风险，保证行程和报价一致性。联动是"提醒"而非"自动改价"——系统不自动修改价格数字（因为价格需要人工确认），只标记哪些行需要关注。

#### 4.2.5 变更备注智能生成

行程报价单调整后保存新版本时，系统**自动检测变更内容**并生成建议备注：

**检测逻辑**：
- 对比当前版本与上一版本的差异（行程区块+报价区块均检测）★（v0.9 扩展：报价变更也纳入检测）
- 识别变更类型（新增/删除/修改 Day、酒店/餐饮/交通/景点变更、天数变化、报价行价格变更、车型方案变更等）★
- 自动生成结构化建议备注

**建议备注示例**：
```
系统检测到以下变更：
☑ D3 酒店：九寨沟假日酒店 → 九寨沟希尔顿（价格 +800/晚）
☑ D5 景点：删除购物点，改为自由活动
☑ 门票报价行：¥3,000 → ¥2,500（Day5景点变更联动更新）★
☐ D7 用餐：团餐 → 自助餐
☐ 新增 D8：返程日
☐ 包车报价行：7座方案 ¥4,000 → ¥4,800（天数增加联动更新）★

请在上方勾选需要记录的变更，也可在下方手工补充：
┌─────────────────────────────────────────────┐
│ （手工输入框，可补充系统未检测到的备注）         │
└─────────────────────────────────────────────┘
```

**操作方式**：
1. 系统自动勾选检测到的变更项（用户可取消勾选）
2. 用户可手工补充额外备注
3. 至少需要保留一项备注（勾选项或手工输入），否则不允许保存
4. 最终备注 = 勾选项（系统生成）+ 手工补充（用户输入），合并存储

> **设计目标**：减少用户每次手写备注的负担，同时保证变更记录的准确性和结构化。★（v0.9 扩展：行程变更和报价变更统一检测，联动更新的报价行也自动出现在建议备注中）

| 价格层级 | 谁填写 | 谁可见 | 说明 |

选择"按百分比加成"（需手动切换）：
  百分比：[ 10 ]%  → 加价 = 旅行社报价 × 10%

选择"按固定金额"（默认）：
  金额：[ ¥3,000 ]  → 加价 = ¥3,000

（两种方式二选一，不可同时使用，默认为"按固定金额"）
```

#### 4.2.10 价格计算逻辑

- **自行模式**：一手地接社自行填写成本价（含分车型）
- **协作模式** ★（v0.9 更新）：成本价由省地接社通过 H5 提交（含分车型），一手地接社收到后在行程报价单内确认使用
- **一手利润**由一手地接社设置（百分比或金额二选一）
- **对旅行社报价** = 成本小计 + 一手利润（系统自动计算，按车型分别计算）
- **旅行社加价**由旅行社销售设置（百分比或金额二选一）
- **对游客报价** = 对旅行社报价 + 旅行社加价（系统自动计算）
- 一旦点击"生成报价快照"，所有数据冻结
- 后续如需改价：必须创建新版本 + 新报价快照
- ★（v0.9 新增）行程修改后报价行联动标记"需更新"，用户更新后触发版本+1
- ★（v0.9 新增）改行程或改报价都触发共享版本号+1

#### 4.2.11 核心操作

- 基于模板创建行程报价单
- 选择行程模式（自行 / 协作）
- **在行程区块编辑行程**（自动保存为新版本，系统智能生成建议备注+手工补充）
- **在报价区块编辑报价**（填写成本价/设置利润/查看自动计算的旅行社报价）★（v0.9 合并）
- **添加一天**：在最后一天（Day N）下方点击"＋ 添加一天"按钮，自动生成 Day N+1，**关联报价行自动标记"需更新"** ★
- **行程修改→报价联动** ★（v0.9）：行程修改后关联报价行自动标红，提醒同步更新
- 版本对比（v2 vs v1，标红差异，**含行程diff+报价diff** ★）
- 设定"当前生效版本"
- 复制任意历史版本作为新基线
- 切换行程模式（自行 ↔ 协作，自动记录）
- **协作模式下"发送成本询价"** ★（v0.9 新增）：
  - 在行程报价单页面点击"发送成本询价"
  - 系统生成**成本询价 H5 链接**（带客户+需求+版本标注）
  - 链接+摘要自动复制到剪贴板
  - 用户粘贴到微信发给省地接社
  - 省地接社在 H5 内填写成本价→"保存并通知"→一手地接社收到
  - 一手地接社收到后在行程报价单内确认使用省地接社提交的成本价
- **"保存并通知"** ★（v0.9 合并版）：
  - 保存行程报价单新版本（含变更备注确认）
  - 自动生成**一条 H5 链接含行程+报价** ★（不再分开生成两个链接）
  - 链接+摘要**自动复制到剪贴板**
  - 页面展示微信卡片预览 + "已复制，去微信粘贴"提示
  - 用户直接切换到微信粘贴发送给境外旅行社
  - 系统自动在沟通记录中心记录"保存并通知"行为
  - 如有"需更新"的报价行未处理，弹出确认提示
- **生成报价快照**：冻结当前版本的所有价格数据，不可修改
- **导出 PDF**：支持旅行社版/游客版/内部版 + 多语言选项

#### 4.2.12 行为规则

- 每次"保存为新版本"必须确认变更备注（**系统建议+手工补充混合模式**），变更检测范围含行程+报价 ★
- **改行程或改报价都触发共享版本号+1** ★（v0.9）
- 任何已生成的报价快照所引用的版本不可再修改，只能基于其新建版本
- 旧版本只读，不允许删除
- 每次版本变更自动生成 H5 链接 + 自动在沟通记录中心生成记录
- 自行模式下，一手地接社运营直接填写成本价
- 协作模式下，省地接社通过 H5 填写成本价，一手地接社收到后确认使用 ★（v0.9）
- **"保存并通知"生成的 H5 链接包含行程+报价协作功能** ★（v0.9 合并）：旅行社打开后可查看行程+报价详情+提交反馈修改意见（可同时关联行程天和报价行）；一手地接社打开后可查看反馈+直接修改行程和报价
- **同一 H5 链接根据登录身份展示不同操作能力** ★（v0.9 更新）：
  - 旅行社销售视角：查看行程+报价（按权限显示对应价格层级）+ 提交反馈修改意见（可关联行程天和报价行）+ 确认报价 + 加价生成游客报价
  - 一手地接社运营视角：查看行程+报价（可编辑）+ 查看旅行社反馈（标注在对应天数行和报价行）+ 修改行程和报价 + 保存并通知
  - 省地接社运营视角 ★（v0.9.1 更新）：查看行程需求（可编辑，可以修改行程内容）★ + 编辑行程（修改酒店/景点/餐饮/交通）★（v0.9.1 新增） + 反馈行程规划（提交修改建议）★（v0.9.1 新增） + 填写成本价（按车型）+ 保存并通知（通知一手地接社）

#### 4.2.13 输出与分享

- **一键生成 PDF**：
  - 旅行社版 PDF：含行程 + 对旅行社报价（按车型），**不含成本价和一手利润**
  - 游客版 PDF：旅行社加价后生成，含行程 + 对游客报价（选定车型），**不含旅行社报价、成本价、利润**
  - 一手地接社内部版：含全部5级价格（含成本价）
- **PDF 多语言导出**：

| PDF 版本 | 内容 | 适用场景 |
|---|---|---|
| 中文版 | 全中文行程+报价 | 默认输出，地接社内部使用 |
| 泰语版 | 全泰语行程+报价 | 游客母语为泰语 |
| 中泰双语对照版 | 双语并行排版（一行中文 + 一行泰语，逐行对照） | 游客确认行程（推荐） |

| 其他语言 | 按需扩展 | 韩语/日语等 |

> **多语言实现方式**：
> - 行程内容默认中文填写，不强制中英双填
> - AI 翻译引擎：采用腾讯云 AI 翻译（TMT 文本翻译）
> - 导出 PDF 时选择目标语言 → 系统调用 AI 翻译 → 渲染对应语言版本
> - 关键信息支持人工校对修改后锁定
> - 双语对照版：双语并行排版——每一行中文下方紧跟一行对应翻译
> - 翻译记忆：已校对锁定的内容进入翻译记忆库，不强制预录入，自然积累

- **生成带标注的 H5 链接** ★（v0.9 合并版：一条链接含行程+报价），粘贴到微信分享
- 链接展示的是"对旅行社报价"版本（旅行社视角）或"对游客报价"版本（旅行社转发给游客时）
- ★（v0.9 新增）协作模式下可生成"成本询价 H5 链接"发给省地接社（详见 4.4）
- 历史报价快照永久保留
- 一手地接社内部可导出含成本价的完整版 PDF

- 三条路径共享同一详情页，仅头部面包屑不同，降低认知负担

### 4.4 H5 链接标注分享与微信协作闭环 (M3) ★（v0.9 大幅扩展）

#### 4.4.1 设计目标

微信是主沟通通道，系统不替代微信。系统为每个关键对象生成带标注的专属 H5 链接，用户一键复制粘贴到微信群，对方点开即可在 H5 页面查看详情、操作、备注。

★（v0.9 核心扩展）：**微信驱动三方协作闭环**。H5 页面是协作面，微信是通知通道。
- **闭环A（你 ↔ 旅行社）**：行程+报价协作——一手地接社"保存并通知"生成含行程+报价的H5链接→微信粘贴→旅行社微信内打开H5→查看行程+报价+反馈（可同时标注行程天和报价行）→复制链接通知→一手地接社微信内H5修改→保存并通知→循环
- **闭环B（你 ↔ 省地接社）** ★（v0.9 新增）：成本询价协作——一手地接社"发送成本询价"生成成本询价H5链接→微信粘贴→省地接社微信内打开H5→查看行程需求+填写成本价→"保存并通知"→一手地接社收到成本报价→在行程报价单内加利润→生成对旅行社报价

#### 4.4.2 链接类型

| 链接类型 | 触发时机 | 链接内容 | 通知方式 |
|---|---|---|---|
| 客户档案链接 | 创建客户时自动生成 | 客户基础信息 + 状态 + 关联行程报价单列表 | 手动复制粘贴微信 |
| **行程报价协作链接** ★（v0.9 合并） | **行程报价单"保存并通知"时自动生成** | **该版本行程+报价详情 + 修改备注 + 旅行社反馈区（行程+报价）+ 一手地接社编辑入口（按身份）** | **"保存并通知"自动复制到剪贴板，用户粘贴到微信** |
| **成本询价H5链接** ★（v0.9 新增） | **协作模式下"发送成本询价"时自动生成** | **行程需求（可编辑）+ 成本价填写区（按车型分列）+ 省地接社"保存并通知"入口** | **自动复制到剪贴板，用户粘贴到微信发给省地接社** |
| 行程版本链接 | 每次保存新版本时自动生成 | 该版本行程详情 + 修改备注 + diff 摘要 | 手动复制粘贴微信 |
| 报价快照链接 | 生成报价快照时自动生成 | **对旅行社报价详情（按车型）+ 行程 + 有效期** | 手动复制粘贴微信 |
| **游客报价链接** | 旅行社加价后生成 | **对游客报价详情（选定车型）+ 行程**，不含旅行社报价 | 手动复制粘贴微信 |
| 沟通备注链接 | 手动添加备注时可生成 | 备注内容 + 上下文 | 手动复制粘贴微信 |

> **行程报价协作链接 vs 行程版本链接** ★（v0.9 更新）：
> - 行程报价协作链接：通过"保存并通知"生成，**链接+摘要自动复制到剪贴板**，H5 页面包含双向协作功能（旅行社可反馈行程+报价、一手地接社可编辑行程+报价），**一条链接含行程+报价** ★
> - 行程版本链接：系统自动生成的版本记录链接，H5 页面以展示为主，**不自动复制到剪贴板**
> - 两者指向同一行程报价单版本，但入口和交互能力不同

> **成本询价H5链接** ★（v0.9 新增）：
> - 通过"发送成本询价"生成，链接+摘要自动复制到剪贴板
> - H5 页面为**省地接社视角**：只显示行程需求（可编辑）+成本价填写区+行程编辑区+行程规划反馈区
> - 省地接社填写成本价后点击"保存并通知"→成本价提交+通知链接复制到剪贴板→粘贴到微信通知一手地接社
> - 成本询价H5链接不包含任何报价信息（对旅行社报价、利润、游客报价等），省地接社只能看到行程需求和填写成本价

#### 4.4.4 链接访问与登录机制 ★（v0.9.1 简化：邀请制）

**准入流程（安全闭环 + 邀请制）** ★（v0.9.1 简化）：

| 步骤 | 访问者状态 | 系统行为 |
|---|---|---|
| 1. 管理者邀请 | 获得邀请链接 | 管理者发送邀请链接给员工（微信/短信） |
| 2. 微信授权登录 | 获取微信身份 | 首次打开H5完成微信授权登录 |
| 3. 判断归属 | 系统根据邀请链接判断归属 | 自动绑定到对应组织（旅行社/省地接社） |
| 4. 直接获得权限 | 正常使用 | 按邀请时分配的角色获得权限，登录态保持 30 天，邀请一次永久有效 |

> ★（v0.9.1 简化）**取消"待审核"状态**，改为"管理者邀请→员工微信授权→直接获得权限"。不再需要审核流程，首次打开H5完成微信授权登录即获得权限。

> **微信内无感登录**：
> - 旅行社/省地接社在微信中点开 H5 链接 → 微信 OAuth 自动识别身份 → 30天登录态有效 → **无需重复登录** ★（v0.9 扩展：省地接社也适用）
> - 首次打开需微信授权（一键授权，无需输入账号密码）
> - ★（v0.9.1 简化）通过邀请制获得权限后，后续在微信内打开任何 H5 链接均自动识别身份，**零门槛使用**

> **邀请规则** ★（v0.9.1 更新）：
> - 新旅行社/新省地接社首次接入：其管理者账号由一手地接社管理者邀请（一次性）
> - 旅行社员工：由本旅行社管理者邀请
> - 省地接社员工：由本省地接社管理者邀请 ★（v0.9 新增）
> - 邀请一次即可：通过微信授权后永久有效，30 天登录态到期重新授权不需要重新邀请
> - ★（v0.9.1 简化）取消审核流程，管理者邀请→员工微信授权→直接获得权限

> **安全铁律**：
> - 未获邀请的微信账号无法查看任何业务数据
> - 链接被转发给外人 → 外人微信授权后无邀请记录 → 无法访问
> - 一手地接社管理者可管理所有账号
> - 每个微信账号只能绑定一个旅行社或省地接社
> - ★（v0.9 新增）成本询价H5链接只有被分配的省地接社可访问，其他省地接社无法打开

> - **零切换**：三方全程在微信内完成，不需要打开PC端、不需要安装App
> - **零门槛**：微信授权登录一次，30天免登，微信内打开H5自动识别身份
> - **有留痕**：每次"保存并通知"、每次反馈提交、每次成本价提交、每次链接打开都自动记录到沟通记录中心
> - **可追溯**：行程报价单版本 v1→v2→v3 每次修改都有变更备注，反馈意见有版本关联，成本报价有独立版本记录
> - **数据隔离**：旅行社看不到省地接社的成本价，省地接社看不到对旅行社报价，不同省地接社之间互相隔离 ★

### 5.1 端到端主流程 ★（v0.9 升级）

```

  → 旅行社销售复制链接，粘贴到微信群
  → 一手地接社运营收到通知，查看 v1 + 初步行程需求 ★（v0.9.1 新增：作为行程规划输入参考）

  → 点击卡片进入【行程报价单】★（v0.9 合并页面）
  → 选择行程模式（自行优先 / 协作模式）

  → 一手地接社在行程报价单页面编辑行程+填写成本价+设置利润
  → 系统按车型分别计算对旅行社报价
  → 点击"保存并通知" ★（v0.9：一条H5链接含行程+报价）
      → 系统生成行程报价协作H5链接+摘要，自动复制到剪贴板

  【协作模式路径】★（v0.9 新增）
  → 一手地接社分配省地接社
  → 在行程报价单页面点击"发送成本询价"
      → 系统生成成本询价H5链接+摘要，自动复制到剪贴板
      → 一手地接社粘贴到微信发给省地接社
  → 省地接社在微信内打开H5 → 查看行程需求+填写成本价（分车型）
      → 点击"保存并通知" → 成本价提交+链接复制到剪贴板
      → 省地接社粘贴到微信通知一手地接社
  → 一手地接社收到成本价 → 确认使用 → 加利润 → 生成对旅行社报价
  → 点击"保存并通知" → 一条H5链接含行程+报价 → 粘贴到微信发给旅行社

  → 旅行社在微信内打开H5 → 查看行程+报价
      → 【如有修改意见】在H5页面提交反馈（可关联行程天+报价行）★
          → 点击"复制链接通知一手地接社" → 粘贴到微信

              → H5内修改行程+报价（行程修改后报价区块联动标记"需更新"）★
              → "保存并通知" → 粘贴到微信
          → 循环直到旅行社确认
      → 【确认行程+报价】进入旅行社加价流程

  → 旅行社在 H5 页面查看分车型报价方案对比
      → 选择确认车型方案（如选9座）

          → 生成游客报价快照
      → 可下载游客版 PDF（含行程+游客报价，不含旅行社报价/成本价/利润）

      → 一键转发摘要到微信

  → 一手地接社可查看旅行社加价金额和游客报价（掌握成交价）
  → 多路线看板更新该路线状态为"已确认" ★

  → 游客反馈修改
  → 【简单事项】→ 微信直接聊确认 → 可选手动备注

  → ...循环...

  → 游客确认 → 旅行社在系统确认 → 状态变更自动记录
  → 导出多语言 PDF 给游客确认

### 5.4 旅行社加价流程

```
[旅行社收到一手地接社报价]
  → 旅行社销售在 H5 页面查看行程+报价
  → 查看分车型方案对比（7座/9座/14座各自报价）
  → 选择确认车型方案（或暂不选择，全部报给游客）
  → 点击"加价生成游客报价"
  → 选择加价方式（默认"按固定金额" ★v0.9.1）：
      ├── 按固定金额（默认）：输入金额（如¥3,000）→ 加价 = ¥3,000
      └── 按百分比加成（需手动切换）：输入百分比（如10%）→ 加价 = 旅行社报价 × 10%
  → 系统自动计算对游客报价 = 旅行社报价 + 加价
  → 生成游客报价快照（冻结，不可修改）
  → 可选操作：
      ├── 生成游客版 H5 链接 → 转发给游客
      ├── 下载游客版 PDF（不含旅行社报价/成本价/利润）
      └── 一键转发摘要到微信

  → 系统自动通知一手地接社："旅行社已加价生成游客报价"
  → 一手地接社可查看：加价方式、加价金额、游客报价
  → 多路线看板更新该路线待办为"已确认" ★
  → 沟通记录中心自动记录加价行为

  若游客要求改价：
  → 旅行社重新加价 → 生成新的游客报价快照
  → 原快照保留（历史追溯）
```

  → 邀请一次永久有效，30天登录态到期重新授权不需要重新邀请

[一手地接社管理者（超级管理员）]
  → 可越级管理所有组织的任何账号

### 5.6 双通道沟通示例

```
场景：游客反馈"Day3酒店太差了，换个好点的，但不要太贵"

【简单事项路径】（微信直接聊）
  → 旅行社微信转达给一手地接社："客户说Day3酒店要换好的，但别太贵"
  → 一手地接社微信问省地接社（协作模式）："Day3换个好点的酒店，成本加多少？"
  → 省地接社微信回复："加300/晚，XX酒店"
  → 一手地接社微信发旅行社："已换XX酒店，新报价+300，v2"
  → 旅行社加价后发游客
  → 可选手动在系统备注留痕

【复杂事项路径】（H5系统交互，推荐）★（v0.9）
  → 旅行社在H5页面提交反馈：
      关联行程Day3 + 报价酒店行："Day3酒店太差，换好点的，但预算控制在¥800/晚以内"
  → 旅行社点击"复制链接通知一手地接社" → 粘贴到微信
  → 一手地接社微信内打开H5 → 查看反馈（标注在Day3行程行+报价酒店行）
  → 一手地接社H5内修改：Day3酒店改为XX酒店（¥800/晚）
      → 报价酒店行联动标记"需更新"→更新价格
  → 点击"保存并通知" → v2链接复制到剪贴板 → 粘贴到微信通知旅行社
  → 旅行社查看v2 → 确认或再反馈

  优势：反馈有结构化记录（关联行程天+报价行），修改有版本追溯，不遗漏
```
