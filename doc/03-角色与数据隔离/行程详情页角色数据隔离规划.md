# 行程详情页三角色视角与数据隔离规划

> 针对当前境外旅行社视角错误出现「省地接社协作」区块和 PandaKing 成本/利润列的问题，统一规划后实施。

## 1. 根因定位（已验证）

- `frontend/src/stores/auth.ts` 中 `currentRole` 默认值写死为 `'pandaking'`，且页面刷新后只从 `localStorage` 恢复 `user` 和 `token`，**没有恢复 `currentRole`**。
- `App.vue` 顶部显示用的是 `user.value.role`（正确显示“境外旅行社”）。
- `RouteDetail.vue` 的 `role = computed(() => auth.currentRole)` 拿到的是默认 `pandaking`。
- 结果：`isPk=true`、`isAgency=false`，导致：
  - 省地接社协作卡（`v-if="isPk"`）被渲染出来。
  - `QuoteTable` 按 PandaKing 角色渲染列（成本① / 利润① / 报价A），而不是 agency 的报价A（成本）/ 利润② / 对客价。

## 2. 修复方案

### 2.1 根因修复：auth store 的 currentRole 必须与 user.role 强一致

将 `currentRole` 改为 `computed(() => user.value?.role ?? 'pandaking')`：
- 刷新后 `user` 从 localStorage 恢复，`currentRole` 自动同步。
- `loadMe()` 更新 `user` 后，`currentRole` 自动同步。
- `dev-login` 切换角色后，`setSession` 更新 `user`，`currentRole` 自动同步。
- 消除“显示角色与内部角色不一致”这一类 bug。

### 2.2 角色布局门控（已存在，修复后自动生效）

| 功能区块 | 出现角色 | 当前条件 | 说明 |
|---|---|---|---|
| 省地接社协作卡 | 仅 pandaking | `v-if="isPk"` | 已正确，修复角色后 agency 不再看到 |
| 协作记录 / 价格回调链路 | 仅 pandaking | `v-if="isPk"` | 已正确 |
| 反馈建议输入框 | agency / provincial | `v-if="isAgency \|\| isProv"` | 已正确 |
| 报价明细表 | 三角色 | `<QuoteTable :role="role" />` | 组件内部按角色渲染不同列 |
| 状态流转 | 三角色（后端过滤） | 无角色门控 | 后端状态机拒绝非法操作，前端仅展示当前状态可用动作 |
| 客户与版本 | 三角色 | 无 | 只读信息，无需隔离 |

### 2.3 报价数据隔离（后端已兜底 + 前端组件渲染隔离）

后端 `role-visibility.ts` 与 `saveVersion` 已按角色隔离：
- `pandaking`：全见 `cost1/profit1/quoteA/profit2/guestPrice`。
- `agency`：仅见 `quoteA`（作为成本）、`profit2`、`guestPrice`；后端保存时只合并 `profit2`，不改动 PandaKing 的成本/利润。
- `provincial`：仅见 `cost1`；通过省地接社 H5 或控制台提交成本。

前端 `QuoteTable` 按 `role` 渲染列：
- `pandaking`：成本① / 利润① / 报价A / 利润②预览 / 对客价。
- `agency`：报价A（成本） / 利润② / 对客价。
- `provincial`：成本①（可填）。

## 3. 验证计划

1. 构建前端，确认无 TS 错误。
2. 部署到 CloudBase 静态托管。
3. 用系统 Chrome + Playwright 实跑线上：
   - agency 登录 → 进入详情页 → 确认：不出现「省地接社协作」、报价表列标题为「报价A（成本）/ 利润② / 对客价」、无成本①/利润①列。
   - pandaking 登录 → 进入同一页 → 确认：出现「省地接社协作」、报价表列标题为「成本① / 利润① / 报价A」。
   - provincial 登录（如可进控制台）→ 确认：只出现成本①列。
4. 强刷后复测，确认无缓存问题。
