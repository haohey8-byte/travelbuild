## 二、与 PRD 存在偏差的 5 处（已纠正 1 处）

### 1. 省地接社是否有控制台路线视图 —— 已纠正

- **PRD 权限矩阵**：省地接社 **无路线视图**，仅通过「成本询价」交互。
- **偏差实现**：`findAll`/`findOne`/`assertVisible` 曾支持 `provincialId` 可见，导致省地接社登录后能看到被分配的路线。
- **已纠正**：`eb48ada` 将 `provincial` 角色控制台路线访问全部关闭；前端导航和 RouteKanban 同步增加拦截。
- **保留**：H5 成本询价、H5ProvincialRoute 仍通过 token 正常工作。

### 2. 缺少 Agency（机构）模型

- **PRD**：境外旅行社 / 省地接社都是「机构」，机构有名称、编号、角色类型。
- **当前实现**：只有 `User.agencyId` 和 `Route.agencyId` 的**字符串**裸值，没有 `Agency` 表。
- **后果**：
  - 一手创建路线时无法下拉选择已有的境外旅行社。
  - 机构名称重复、编号容易写错。
  - 无法维护机构档案（联系方式、备注等）。
  - 无法做机构级统计。

### 3. 创建路线时的数据归属混乱

- **PRD 数据流**：
  - 一手创建路线：填写客户信息，并**指定/选择境外旅行社**；路线归属该旅行社（`agencyId = 该旅行社机构编号`）。
  - 境外旅行社创建路线：自动归属本机构（`agencyId = 当前用户 agencyId`）。
  - 省地接社：不创建路线。
- **当前实现**：
  - 创建弹窗让任何人填「旅行社」文本，但不校验、不关联机构编号。
  - 一手创建时如果 agencyId 为空，则该路线对任何旅行社都不可见（除非降级到 pandaking）。
  - 旅行社创建时若 token 中 agencyId 为空，则创建出的路线也对自己不可见。

### 4. 顶部「角色切换」select 不符合真实账号体系

- **PRD**：一个账号固定一个角色，通过邀请绑定机构与层级。
- **当前实现**：App.vue 顶部提供 select 切换「一手 / 旅行社 / 省地接社」，每次切换走 `dev-login` 换 seed 账号。这只是一个**演示快捷入口**，不是真实账号管理。
- **已纠正**：移除顶部 select，改为显示当前账号名称+角色，切换账号需退出重新登录。

### 5. 缺少机构管理 / 账号设置页面

- **PRD**：账号管理模块包含「成员列表、角色权限、邀请、停用」。
- **当前实现**：账号页有成员列表、邀请、权限矩阵，但：
  - 没有「机构列表/机构管理」（因为没 Agency 表）。
  - 一手邀请机构管理员时，需要手动输入机构编号，没有下拉选择。
  - 没有微信 OAuth 真实接入（仍用 dev-login 演示）。

## 三、改造方案（按优先级）

### P0：确保当前隔离正确（已完成）
- `provincial` 角色不进入控制台路线管理。
- 前端按角色过滤导航。
- 移除顶部角色切换 select。

### P1：建立 Agency 模型并控制创建路线归属
1. **新增 `Agency` 表**：`id`（机构编号）, `name`, `role`（agency/provincial）, `createdAt`。
2. **数据库迁移**：把现有 `User.agencyId` / `Route.agencyId` / `CostInquiry.provincialId` 字符串映射为 `Agency.id`。
3. **Seed 写入**：新增 `Agency` 种子（环球旅行社、川内地接社）。
4. **创建路线 API 改造**：
   - 一手创建：必填 `agencyId`（下拉选择 Agency 中 role=agency）。
   - 旅行社创建：自动取当前用户 `agencyId`。
   - 后端校验：如果当前用户 role=agency，传进来的 agencyId 必须等于自己的 agencyId（防伪造）。
5. **前端创建路线弹窗**：
   - 一手显示「选择境外旅行社」下拉（读取 Agency 列表）。
   - 旅行社隐藏该字段，自动使用本机构。
   - 移除对所有人开放的「旅行社」文本输入。

### P2：账号管理增强
1. 新增 `/api/agencies` 接口：一手可增删改查 Agency；机构用户只读本机构。
2. 前端账号页：
   - 一手显示「机构管理」卡片。
   - 邀请机构管理员时，从 Agency 列表下拉选择，而不是手填编号。
3. 账号页显示「当前机构」信息。

### P3：真实登录（可选，不影响本轮）
- 接入微信 OAuth2：`/api/auth/wechat` / `/api/auth/wechat/callback`。
- 生产环境关闭 `dev-login`。
- 当前保留 dev-login 作为演示入口。

## 四、各角色登录后的数据流（改造后）

### 一手 PandaKing
- 看板：见全部路线（不区分机构）。
- 创建路线：选择客户信息 + 选择境外旅行社 → 路线 `agencyId` = 该旅行社。
- 路线详情：可见成本①、成本②、加价、游客价；可发送成本询价、分配省地接社。
- 账号：可管理机构、邀请机构管理员、查看成员、停用账号。

### 境外旅行社（admin / staff）
- 看板：仅见 `agencyId = 本机构` 的路线。
- 创建路线：自动归属本机构，状态为「咨询中」；提交草案后转「待一手确认」。
- 路线详情：仅见客户档案、行程草案、加价、游客价；**不可见成本①/成本②**。
- 账号：admin 可邀请本机构 staff；只读本机构成员。

### 省地接社（admin / staff）
- 看板：**无**（控制台不显示路线管理）。
- 账号页：显示「我的成本询价」，点击 H5 链接填写成本①。
- H5 协作：通过一手发来的 `provincial-route/:token` 编辑被分配路线的行程。
- 无法看到任何旅行社信息、客户档案、报价加价。
