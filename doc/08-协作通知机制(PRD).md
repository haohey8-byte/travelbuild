# 08 · 协作通知机制（微信沟通 + 结构化 H5）

> 本文件细化 PRD v0.9.11 中「协作通知」部分，定义各角色如何通过「复制粘贴协作 H5 到微信群」完成通知。
> 对应实现：`frontend/src/utils/share.ts`（文案构造）、`RouteDetail.vue`（控制台）、`H5Route.vue`（H5 页）。

## 1. 设计初衷

- **协作发生在微信群**：客户、一手 PandaKing、境外旅行社的日常沟通都在微信，强依赖群消息触达。
- **但纯微信聊天是非结构化的**：方案版本、报价、反馈意见散落在对话里，容易丢失、难追溯、跨角色口径不一致。
- **因此采用「微信沟通 + 结构化 H5」双轨**：
  - 所有**正式信息**（行程、5 级报价、版本、反馈记录）沉淀在**协作 H5**（版本化、可回溯、带 OG 分享卡片）。
  - **微信群只做「通知 + 轻量讨论」**，不承载正式数据。
- **通知方式（MVP）**：复制粘贴 H5 链接 / 结构化通知文案到微信群。
  - 不依赖企业微信机器人，零额外集成成本；任何角色、任何手机都能用。
  - 复制动作由前端完成（Clipboard API，非安全上下文退化 `execCommand`），**无需后端改动、无需重部署后端**。

## 2. 角色与触发点（每个角色的「规划提交」「反馈意见」都走复制 H5）

| 角色 | 入口 | 规划提交（动作 → 复制通知） | 反馈意见（→ 复制通知） |
|---|---|---|---|
| 一手 PandaKing | 控制台 RouteDetail | 保存并通知（生成H5）/ 确认采用 / 加价 / 游客确认 / 付款成单 | 回传反馈 |
| 境外旅行社 | 控制台 RouteDetail | 修订重交 / 发报价 v1 | （回传反馈由一手在确认环节操作） |
| 客户 / 协作方 | H5 页 H5Route | 查看方案时「复制协作链接发到微信群」 | 提交修改反馈 |

**原则**：凡是「把方案往前推进一步」的状态流转动作，或「给出修改意见」的反馈动作，系统都自动生成一段可直接粘贴到微信群的通知文案（见 §4）。操作后剪贴板即包含该文案，用户只需「打开微信 → 粘贴 → 发送」。

**例外**：`拒绝 / 流失` 为负向事件，不自动复制通知文案（避免刷屏），但页面仍保留「复制链接」按钮可手动转发。

## 3. 通知文案结构（统一格式）

```
【行程协作·方案更新 | 反馈意见】<客户名> · <目的地>
<操作方> <事件名>
「<反馈建议 / 备注>」（仅反馈意见类显示）
👉 查看并回复：<协作 H5 链接>
```

- **方案更新类（规划提交）**：只含 `事件名`（如「修订重交」「确认采用」「游客确认」）。
- **反馈意见类**：含 `事件名` + 「反馈建议」引用块。
- `<操作方>` 由 `roleLabel()` 标注：一手 PandaKing / 境外旅行社 / 省地接社；有登录名时显示登录名。
- `<协作 H5 链接>` 指向后端 SSR 分享页 `/share/route/:token`（带 OG 注入，微信内展开为带客户名+目的地的卡片）。

## 4. 实现要点

- `utils/share.ts`
  - `collabNotifyText(opts)`：统一构造规划提交 / 反馈意见两类通知文案。
  - `roleLabel(role)`：角色 → 中文标签。
  - `shareH5Url(token)` / `shareH5Caption(route)`：链接与「客户·目的地 定制行程报价」说明。
  - `copyText(text)`：剪贴板复制（带退化方案）。
  - `feedbackNotifyText(opts)`：保留的兼容封装（旧反馈文案格式）。
- 控制台 `RouteDetail.vue`
  - `onAction`：除 `reject` 外，所有状态流转动作成功后生成并复制 `collabNotifyText` 通知，页面内显示「通知文案」卡片 + 「再复制」。
  - 新增「复制链接」按钮：随时把当前方案（说明 + 链接）复制到剪贴板转发。
- H5 页 `H5Route.vue`
  - 提交反馈成功后生成并复制 `collabNotifyText` 通知。
  - 页面常驻「📋 复制协作链接发到微信群」按钮，供查看方随时转发方案到群。

## 5. 进阶（未做，可排期）

- **企业微信群机器人 Webhook**：把通知文案自动 POST 到群，免去人工粘贴。需群机器人地址（敏感信息，建议走环境变量 / 密钥管理，勿落前端）。
- 动态 OG 分享卡片已具备（后端 `/share/route/:token` 注入客户名+目的地+人数+日期+对客价），分享即结构化预览。

## 6. 验收标准

- [ ] 每个角色的规划提交 / 反馈动作后，剪贴板含 §3 格式文案 + 有效 H5 链接。
- [ ] 任意手机微信粘贴后，链接展开为带客户名 / 目的地的卡片。
- [ ] 反馈双向可见：H5 提交 ↔ 控制台回传，且各自动作都触发群通知文案。
- [ ] 非安全上下文（http）下复制仍可用（execCommand 退化）。
