# 08 · 协作通知机制（微信沟通 + 结构化 H5）

> 本文件是「协作通知机制」的**当前权威 PRD**（原 v0.9.11 细化，已随多轮协作模型、微信可打开链接修复同步更新到 2026-07）。
> 对应实现：`frontend/src/utils/share.ts`（文案构造 + 差异计算）、`RouteDetail.vue`（控制台）、`H5Route.vue`（对客 H5）、`H5ProvincialRoute.vue`（省地接社协作 H5）。
> 关键提交：`60063a5`（微信链接改指向后端免登录 SSR 页）、`73024ae`（省地接社按钮合并）、`fa7f54f`（多轮协作 + 关键变更摘要）。

## 1. 设计初衷

- **协作发生在微信群**：客户、一手 PandaKing、境外旅行社的日常沟通都在微信，强依赖群消息触达。
- **但纯微信聊天是非结构化的**：方案版本、报价、反馈意见散落在对话里，容易丢失、难追溯、跨角色口径不一致。
- **因此采用「微信沟通 + 结构化 H5」双轨**：
  - 所有**正式信息**（行程、5 级报价、版本、反馈记录）沉淀在**协作 H5**（版本化、可回溯、带 OG 分享卡片）。
  - **微信群只做「通知 + 轻量讨论」**，不承载正式数据。
- **通知方式（MVP）**：复制粘贴 H5 链接 / 结构化通知文案到微信群。
  - 不依赖企业微信机器人，零额外集成成本；任何角色、任何手机都能用。
  - 复制动作由前端完成（Clipboard API，非安全上下文退化 `execCommand`），**无需后端改动、无需重部署后端**。
  - **通知链接必须公开可访问（免登录）**：微信内点击的是后端 SSR 分享页 `/share/route/:token`（带 OG 卡片），**不是**需登录的控制台详情页。早期版本误指向控制台导致微信内打不开，已修复（commit `60063a5`）；前端 `.env.production` 注入后端域名，`shareH5Url(token)` 统一产出该公开链接。

## 2. 角色与触发点（每个角色的「规划提交」「反馈意见」都走复制 H5）

| 角色 | 入口 | 规划提交（动作 → 复制通知） | 反馈意见（→ 复制通知） |
|---|---|---|---|
| 一手 PandaKing | 控制台 RouteDetail | 保存并通知（生成H5）/ 确认采用 / 加价 / 游客确认 / 付款成单 | 回传反馈 |
| 境外旅行社 | 控制台 RouteDetail | 修订重交 / 发报价 v1 | （回传反馈由一手在确认环节操作） |
| 客户 / 协作方 | H5 页 H5Route | 查看方案时「复制协作链接发到微信群」 | 提交修改反馈 |
| 省地接社 | 协作 H5 H5ProvincialRoute | 每轮「保存并回传一手」（行程 + 成本① + 可选说明） | （说明随回传一并提交，不另起反馈流） |

**原则**：凡是「把方案往前推进一步」的状态流转动作，或「给出修改意见」的反馈动作，系统都自动生成一段可直接粘贴到微信群的通知文案（见 §4）。操作后剪贴板即包含该文案，用户只需「打开微信 → 粘贴 → 发送」。

**例外**：`拒绝 / 流失` 为负向事件，不自动复制通知文案（避免刷屏），但页面仍保留「复制链接」按钮可手动转发。

## 3. 通知文案结构（统一格式）

```
【行程协作·方案更新 | 反馈意见】<客户名> · <目的地>
<操作方> <事件名>
「<反馈建议 / 备注>」（仅反馈意见类显示）
【本轮关键变更】（仅省地接社多轮回传时显示，见 §3.1）
  基于 v<上一版号>
  成本① 合计 ¥<旧> → ¥<新>（<n>项变更）
  • <项目名>：¥<旧> → ¥<新> / (新增) ¥<新> / (移除)
  行程：<旧天数> → <新天数> 天
  • D<天> <城市>→<城市> / (<新增>) / (<移除>)
👉 查看并回复：<协作 H5 链接>
```

### 3.1 省地接社 ↔ 一手「多轮反复沟通」与关键变更摘要

> 协作模型修正（用户明确纠正）：省地接社与一手之间**不是一次性回传**，而是多轮「派单 → 回传 → 一手反馈 → 再修订回传」的循环，直到一手确认。

- **痛点**：每一轮，一手在微信里只收到"省地接社 更新了行程规划并回传成本 + 链接"，必须点开逐页比对才知道本轮改了哪些价/行程——多轮下沟通成本爆炸，且易漏看价格变动。
- **决策**：每一轮回传的微信通知文案**必须自动携带「本轮关键变更」增量块**（成本①前后对比 + 逐项变更、行程天数与逐天城市变更），使一手无需点开即可获知本轮增量。
- 该摘要由前端 `share.ts` 的 `diffProvincialChanges()` 在保存时基于「加载基线快照 vs 当前编辑」计算，向后兼容（`collabNotifyText` 不传 `changes` 时行为与旧版一致）。

- **方案更新类（规划提交）**：只含 `事件名`（如「修订重交」「确认采用」「游客确认」）。
- **反馈意见类**：含 `事件名` + 「反馈建议」引用块。
- `<操作方>` 由 `roleLabel()` 标注：一手 PandaKing / 境外旅行社 / 省地接社；有登录名时显示登录名。
- `<协作 H5 链接>` 指向后端 SSR 分享页 `/share/route/:token`（带 OG 注入，微信内展开为带客户名+目的地的卡片）。

## 4. 实现要点

- `utils/share.ts`
  - `collabNotifyText(opts)`：统一构造规划提交 / 反馈意见两类通知文案。
  - `roleLabel(role)`：角色 → 中文标签。
  - `shareH5Url(token)` / `shareH5Caption(route)`：链接与「客户·目的地 定制行程报价」说明。
  - `copyText(text)`：剪贴板复制（带退化方案）。
  - `feedbackNotifyText(opts)`：保留的兼容封装（旧反馈文案格式）。
- 控制台 `RouteDetail.vue`
  - `onAction`：除 `reject` 外，所有状态流转动作成功后生成并复制 `collabNotifyText` 通知，页面内显示「通知文案」卡片 + 「再复制」。
  - 新增「复制链接」按钮：随时把当前方案（说明 + 链接）复制到剪贴板转发。
- H5 页 `H5Route.vue`
  - 提交反馈成功后生成并复制 `collabNotifyText` 通知。
  - 页面常驻「📋 复制协作链接发到微信群」按钮，供查看方随时转发方案到群。
- 省地接社协作 H5 `H5ProvincialRoute.vue`（多轮回传 + 关键摘要）
  - `onSubmitHandoff`：每轮保存行程 + 成本①，并（可选）随回传一并提交说明；由 `diffProvincialChanges()` 算差异、`collabNotifyText({ changes })` 生成带「本轮关键变更」块的通知文案。
  - 加载时记录 `costBaseline` / `itineraryBaseline` 快照，保存时对比出增量。
  - 已移除"仅首次回传成本"守卫（`if (!alreadySubmitted …)`），支持任意轮次改价都回传一手（commit `fa7f54f`）。

## 6. 验收标准

- [ ] 每个角色的规划提交 / 反馈动作后，剪贴板含 §3 格式文案 + 有效 H5 链接。
- [ ] 任意手机微信粘贴后，链接展开为带客户名 / 目的地的卡片。
- [ ] 反馈双向可见：H5 提交 ↔ 控制台回传，且各自动作都触发群通知文案。
- [ ] 省地接社每轮回传后，微信通知文案含「本轮关键变更」块（成本①前后对比 + 行程城市变更），一手无需点开即可获知增量。
- [ ] 省地接社多轮修改价格后，每轮都实际回传最新成本①（无"仅首次"截断）。
- [ ] 非安全上下文（http）下复制仍可用（execCommand 退化）。
- [ ] 微信内粘贴的通知链接可正常打开（指向后端免登录 SSR 页 `/share/route/:token`，非控制台需登录页）。
