# 总览：省地接社协作 / 成本询价 / 旅行社加价 修正（2026-07-15）

## 做了什么
对照 PRD v0.9.11，修正了你提出的三个问题，并补齐了对照发现的差异点，两端编译全部通过。

1. **省地接社协作（纠正"无路线视图"错误）**
   - 路线 now 有 `provincialId`：一手分配后，省地接社**可见该路线并参与行程规划与报价**。
   - agency ↔ provincial **价格与身份物理隔绝**：序列化层剥离跨机构字段（agency 看不到 provincialId；provincial 看不到 agencyId/旅行社名）。
   - 新增「省地接社协作 H5」：可编辑分配给自己的行程 + 提交规划建议，保存后同步一手。

2. **成本询价模块（补上）**
   - 一手发起 → 生成 H5 token → 省地接社填成本① → 一手应用写入路线报价（成本①）。端到端打通。

3. **旅行社加价（修正写入逻辑）**
   - 旅行社保存时**仅改自身加价 markup**，成本①/② 取自上一版并重算对客总价，不再被清零。前端 agency 可编辑加价列。

4. **PRD↔代码差异分析文档**
   - `doc/PRD代码差异分析_2026-07-15.md`：已实现 / 部分实现 / 未实现清单 + 上线步骤 + 待办优先级。
