# 05 · 验收标准

> 每个模块的可验证信号。阶段评审（Quality Review 闸门）逐项核对。

## § 路线管理（核心，必须全过）
- [ ] 看板/列表/详情三视图可切换，数据一致
- [ ] 旅行社可发起结构化行程草案并提交一手确认
- [ ] 一手在「待一手确认」可见 [确认采用 / 回传修改反馈]，状态正确流转
- [ ] 5 级报价引擎：车型/住宿/门票/餐饮/其他，成本①与成本②分离存储
- [ ] 游客报价 = 成本① + 成本② + 旅行社加价，计算无误
- [ ] 「保存并通知」生成新 version（v1→v2），草稿/正式分离
- [ ] 编辑自动保存草稿，标「草稿·未通知」
- [ ] 发送成本询价 H5 → 省地接社填成本价 → 一手采纳回填，数据隔离
- [ ] 角色切换：旅行社不可见成本①②；省地接社仅见自身成本价
- [ ] 沟通时间线记录每次修改/通知，可 @引用

## § 知识库
- [ ] 一手可增删改；旅行社只读；省地接社不可见
- [ ] 全文搜索 + 标签筛选可用
- [ ] 条目可关联路线 ID

## § 账号管理
- [ ] 邀请制：仅一手发邀请，链接带角色+有效期
- [ ] 字段级权限矩阵生效（成本①②仅一手；省地接社隔离）
- [ ] 停用账号立即失效 token

## § 案例展示（含合规）
- [ ] 案例仅能由「已确认」路线派生
- [ ] **合规校验**：发布前拦截真名/证件号/精确日期/合同价；校验失败禁止发布
- [ ] 发布人二次确认弹窗必现
- [ ] 已下线案例公开 H5 返回 404
- [ ] 公开 H5 免登录 + EN/ZH/TH/RU

## § 通用（性能/安全/质量）
- [ ] 接口 P95 响应 < 300ms（列表/详情）
- [ ] 关键路径单测覆盖 ≥ 70%
- [ ] 密钥全在 .env，无硬编码
- [ ] E2E 覆盖主链路（草案→确认→报价→发布案例）
- [ ] 部署后健康检查通过，UAT 签字
