# 数据库与部署（CloudBase Postgres）

本文档面向「入境游定制协作工作台」后端，说明如何用 **CloudBase Postgres**（腾讯云 PostgreSQL 实例）起库、迁移、种子，并跑通前后端。

## 1. 准备 CloudBase Postgres 连接串

1. 进入 CloudBase 控制台 →「数据库 / PostgreSQL」创建实例（或与已有实例对接）。
2. 获取连接串，格式形如：
   ```
   postgresql://<user>:<password>@<host>:5432/<dbname>?schema=public&sslmode=require
   ```
   > CloudBase Postgres 默认要求 SSL，务必保留 `sslmode=require`，否则连接会被拒绝。
3. 将连接串写入 `backend/.env` 的 `DATABASE_URL`（模板见 `backend/.env.example`）。

## 2. 执行数据库迁移

项目已用 `prisma migrate diff` 离线生成好初始迁移，位于 `backend/prisma/migrations/0_init/migration.sql`。

**方式 A（推荐，由 Prisma 管理）：**
```bash
pnpm -F backend prisma:deploy      # = prisma migrate deploy，按 migrations/ 顺序应用
```

**方式 B（手动执行 SQL）：**
将 `backend/prisma/migrations/0_init/migration.sql` 在 CloudBase Postgres 控制台/客户端直接执行。

> 说明：`prisma migrate dev` 需要 shadow database，不适合直连生产/云库；云库统一用 `migrate deploy`。

## 3. 写入种子数据

```bash
pnpm -F backend db:seed            # = tsx prisma/seed.ts
```
种子会创建：一手 PandaKing、境外旅行社、省地接社三个演示用户，以及一条演示邀请（`token` 见控制台输出），用于 `dev-login` 与 `accept-invite` 联调。

## 4. 本地开发联调

```bash
# 终端 1：后端（DEV_BYPASS_AUTH=true 免 token 联调）
pnpm -F backend start:dev

# 终端 2：前端（Vite 代理 /api → :3000）
pnpm -F frontend dev
```
浏览器打开 Vite 提示的地址。未登录时点右上角角色下拉即可「开发登录」；路线详情页可「生成协作 H5 链接」并在微信/手机打开 `/h5/route/:token` 只读查看与反馈。

## 5. 生产部署要点

- `backend/.env`：`DEV_BYPASS_AUTH=false`、`NODE_ENV=production`、`JWT_SECRET` 改为随机长串。
- 接入微信：填 `WECHAT_APPID` / `WECHAT_SECRET`，`auth/wechat/callback` 即可用 code 换 openid 真实登录。
- 前端 `pnpm -F frontend build` 产物交由 CDN / EdgeOne 托管；后端由 CloudBase 云托管（Docker）运行。
- 健康检查：`GET /api/health`。

## 6. 一键命令速查

| 目的 | 命令 |
|---|---|
| 生成 Prisma Client | `pnpm -F backend prisma:generate` |
| 云库迁移 | `pnpm -F backend prisma:deploy` |
| 种子 | `pnpm -F backend db:seed` |
| 后端类型检查 | `pnpm -F backend typecheck` |
| 后端测试 | `pnpm -F backend test` |
| 前端类型检查 | `pnpm -F frontend typecheck` |
| 前端构建 | `pnpm -F frontend build` |
