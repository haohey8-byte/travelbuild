# 模块：路线管理（核心）

> 菜单：路线管理。PRD 章节 4.1–4.7。这是 MVP 的「 spine」，其他模块挂靠此数据。

## 1. 用户故事
- **故事 36（升级）**：境外旅行社发起「结构化行程规划草案」（按天：城市/景点/住宿偏好/餐饮偏好 + 预算区间），提交一手确认。
- **故事 37**：一手确认采用草案 → 转正出报价；或回传修改反馈 → 路线转「待旅行社修订」。
- **故事 1**：一手创建路线 + 客户档案，进入行程报价编辑。
- **故事 5**：5 级报价引擎（车型/住宿/门票/餐饮/其他），成本①②分离。
- **故事 12**：发送成本询价 H5 → 省地接社填成本价 → 一手采纳回填。

## 2. 业务规则
- 行程与报价**一体化存储**（一个 version 含 both）。
- 成本①（省地接社成本）与成本②（一手利润）**强制分离字段**。
- 游客报价 = 成本① + 成本② + 旅行社加价（模式 A/B 并存）。
- 每次「保存并通知」生成新 **version**（v1, v2…），草稿与正式版分离。
- 编辑自动保存草稿（draft=true 标「草稿·未通知」）。
- 双向回路：旅行社草案 → 待一手确认 → 确认/反馈 → 待旅行社修订 → 循环。

## 3. 状态机
### 3.1 客户/路线协作状态
```
[咨询中] →(旅行社发草案)→ [待一手确认]
[待一手确认] →(一手确认采用)→ [待报价]
[待一手确认] →(一手回传反馈)→ [待旅行社修订] →(旅行社修订重发)→ [待一手确认]
[待报价] →(发v1)→ [已报价] →(旅行社加价)→ [旅行社已报价] →(游客确认)→ [已确认] →(付款)→ [已成单]
                                  |                              |
                       (超期未回复)→[待跟进]        (明确拒绝)→[已流失]
```
### 3.2 看板/列表筛选状态
全部 / 待一手确认 / 待旅行社修订 / 待报价 / 待反馈 / 待确认 / 已确认 / 已流失（各带计数）。

## 4. 页面与交互
| 页面 | 核心元素 | 操作 |
|---|---|---|
| 看板视图 | 卡片（客户/状态/版本/角标）+ 统计条 | 新建、筛选、打开 |
| 列表视图 | 表格（中英文名/国籍/目的地/人数/日期/状态）+ 搜索 | 按状态筛选、打开 |
| 详情 Tab1 行程报价 | 行程编辑器 + 5 级报价表 + 成本①② | 编辑、发送成本询价、保存并通知 |
| 详情 Tab2 客户信息 | 客户档案 + 行程草案(旅行社) + 版本历史 | 查看、修订草案 |
| 详情 Tab3 沟通时间线 | 沟通记录流 + @引用 | 新增、引用 |
