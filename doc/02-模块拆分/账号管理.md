# 模块：账号管理

> 菜单：账号。PRD 章节 3（角色定义与账号体系）。邀请制，无公开注册。

## 1. 用户故事
- 作为一手管理员，我希望通过邀请链接/邮件添加成员并指定角色，以便控制谁能看成本、谁能发起草案。
- 作为一手管理员，我希望查看字段级权限矩阵，以便向旅行社解释可见性边界。

## 2. 业务规则
- 三种角色：一手 PandaKing / 境外旅行社 / 省地接社。
- 邀请制：仅一手可发邀请，邀请链接带角色与有效期。
- 字段级权限（关键）：
  - 成本①、成本②：仅一手可见。
  - 客户档案、行程草案：一手 + 对应旅行社可见。
  - 省地接社成本价：仅该省地接社 + 一手可见，跨社隔离。
- 停用账号立即失效 token。

## 3. 状态机
```
[ invited ] →(接受邀请)→ [ active ] →(停用)→ [ disabled ]
```

## 4. 页面与交互
| 页面 | 核心元素 | 操作 |
|---|---|---|
| 成员列表 | 表格（姓名/角色/状态/最后活跃） | 邀请、改角色、停用 |
| 角色权限 | 字段级矩阵表 | 只读查看 |
