# 模块：知识库

> 菜单：知识库。PRD 章节 4.7（规划为 MVP 一期）。

## 1. 用户故事
- 作为一手运营，我希望把常用 SOP、话术、资源（酒店/车队/门票）沉淀为可检索条目，以便快速响应旅行社咨询、减少重复沟通。
- 作为一手运营，我希望条目能关联具体路线，以便在报价时直接引用历史方案。

## 2. 业务规则
- 条目含：标题、分类（SOP/话术/资源/目的地知识）、标签、正文（富文本/结构化）、创建人、更新时间。
- 权限：一手可增删改；旅行社**只读**；省地接社不可见（隔离）。
- 支持全文搜索 + 标签筛选。
- 可选关联路线 ID（外键，软引用）。

## 3. 状态机
```
[草稿] →(发布)→ [已发布] →(下线)→ [已下线]
```
（简单生命周期，无强流转约束）

## 4. 页面与交互
| 页面 | 核心元素 | 操作 |
|---|---|---|
| 列表/搜索 | 卡片（标题/分类/标签）+ 搜索框 + 标签筛选 | 新建、打开 |
| 详情/编辑 | 表单（标题/分类/标签/正文）+ 关联路线选择器 | 保存、发布、下线 |
