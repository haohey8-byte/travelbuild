# 模块：案例展示（获客 · 宣传闭环）

> 菜单：案例展示。PRD 章节 4.8（MVP 一期，原 Phase 2 拉回）。**含 4.8.6 合规发布校验**。

## 1. 用户故事
- 作为一手运营，我希望把已确认路线脱敏后发布为案例，以便微信传播获客、沉淀品牌资产。
- 作为潜在客户，我希望免登录浏览公开 H5 案例，以便参考类似行程并发起咨询。

## 2. 业务规则
- 案例**只能由「已确认」路线派生**（不可发布进行中路线）。
- **脱敏硬约束**（4.8.6）：仅可展示 目的地 / 天数 / 主题 / 亮点 / 参考价格区间。
  - **禁止**：客户真名、证件号、精确出行日期、客户专属合同价。
- 发布前强制脱敏校验开关 + 发布人二次确认弹窗。
- 已发布案例可一键下线（下线后公开 H5 404）。
- 公开 H5 免登录，支持 EN/ZH/TH/RU。
- 与路线管理共用「路线」数据脊柱（案例 = 路线发布态派生）。

## 3. 状态机
```
[草稿(派生)] →(合规校验通过+二次确认)→ [已发布] →(下线)→ [已下线]
                                  |                        ↑
                          (校验失败)→ [草稿(打回修正)] ─────┘
```

## 4. 页面与交互
| 页面 | 核心元素 | 操作 |
|---|---|---|
| 案例库 | 卡片（目的地/天数/主题/价格区间）+ 新建(从路线派生) | 发布、下线、分享 |
| 编辑/校验 | 脱敏字段表单 + 合规校验开关 + 二次确认 | 保存、发布 |
| 公开 H5 | 响应式案例页 + 分享(链接/二维码) | 浏览、发起咨询 |
