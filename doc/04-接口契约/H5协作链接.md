# 接口契约：H5 协作链接

## 链接类型
1. **路线协作 H5**（带版本号）：`/api/h5/route/:token` —— 含行程+报价，微信内优化。
2. **省地接社统一协作 H5**（免登录）：`/api/h5/route/:token` —— 当 token 对应的 `RouteShare.role = provincial` 时，省地接社可在同一页面编辑行程并填写成本①。
3. **成本询价 H5**（仅填写成本，已弃用）：`/api/h5/cost-inquiry/:token` —— 保留旧入口，但新流程不再生成。
4. **公开案例 H5**（免登录）：`/api/cases/:id/public` —— 获客展示。
5. **邀请接受 H5**（前端 SPA）：`/h5/invite/:token` —— 受邀者打开后填名称即接受邀请、加入对应机构（复制此链接发到微信群）。

## 端点
| Method | Path | 说明 | 权限 |
|---|---|---|---|
| POST | `/api/routes/:id/share` | 生成协作 H5 token+链接（含当前 version） | 一手/旅行社 |
| GET | `/api/h5/route/:token` | 渲染协作 H5（按 token 解析路线+version+角色） | 持 token |
| POST | `/api/h5/route/:token/edit` | 省地接社协作页提交：编辑行程 + 填写成本① | 持 token |
| POST | `/api/routes/:id/provincial-share` | 一手发起省地接社协作：同时完成分配 + 创建成本询价，返回统一协作 H5 链接 | 一手 |
| POST | `/api/routes/:id/cost-inquiry` | （旧）单独生成成本询价 H5 | 一手 |
| GET | `/api/h5/cost-inquiry/:token` | （旧）渲染询价 H5 | 持 token 省地接社 |
| POST | `/api/invites` | 创建邀请（返回 token，前端拼 `/h5/invite/:token` 链接复制） | 一手/本机构管理员 |
| GET | `/api/invites/:token` | 邀请详情（角色/机构预览，供 H5 页展示） | 持 token |
| POST | `/api/accept-invite` | 接受邀请 → 建账号 + 发 JWT | 持 token |
| POST | `/api/cases` | 从已确认路线派生案例（脱敏校验） | 一手 |
| GET | `/api/cases/:id/public` | 公开案例 H5（免登录） | 公开 |
| POST | `/api/cases/:id/publish` | 发布（过合规校验+二次确认） | 一手 |
| POST | `/api/cases/:id/unpublish` | 下线（H5→404） | 一手 |

## 省地接社协作流程（微信群协作）
1. 一手在路线详情选择省地接社机构，点击「发起省地接社协作」。
2. 后端同时完成：
   - 将路线分配给该省地接社（更新 `Route.provincialId`）
   - 创建一条 `CostInquiry` 记录（状态 pending）
   - 创建一条 `RouteShare`（role=provincial），并关联该 `CostInquiry`
3. 前端生成统一链接 `https://<前端域名>/#/h5/provincial-route/<shareToken>`，复制后发到微信群。
4. 省地接社打开链接：
   - 可编辑行程（城市/景点/住宿/餐饮）
   - 可填写成本①
   - 一次提交同时同步行程与成本
5. 一手在路线详情「协作记录」中查看状态，已回传的成本①可点击「应用成本①」写入路线报价。

## 数据关系
- `RouteShare.costInquiryId` 唯一关联到 `CostInquiry.id`
- 一个 RouteShare 对应一个 CostInquiry（一对一）
- 删除 CostInquiry 时，`RouteShare.costInquiryId` 自动置空（SET NULL）

## 合规校验（案例发布前置）
- 服务端扫描：客户真名/证件号/精确出行日期/合同价 任一命中 → 拒绝发布，返回命中字段。
- 前端二次确认弹窗为 UX 层，服务端为强制层。
