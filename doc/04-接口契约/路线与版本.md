# 接口契约：路线与版本

## 实体：Route
```
Route {
  id, customerName, customerNameCn, country, agency, destination,
  groupSize, travelDate, statusKey,          // 见状态机
  modeKey,                                   // collab | solo
  createdBy, createdAt, updatedAt
}
```

## 版本机制（核心）
- 每次「保存并通知」→ 新建 `RouteVersion`，`version` 自增（v1, v2…）。
- `draft: boolean` —— true 标「草稿·未通知」，不生成对外 H5。
- 一个 Route 含多个 Version；当前生效版 = 最大 version 且 draft=false。
- 行程与报价一体化存于 Version（itinerary JSONB + quote JSONB）。

## 端点
| Method | Path | 说明 | 权限 |
|---|---|---|---|
| POST | `/api/routes` | 新建路线+客户档案 | 一手/旅行社 |
| GET | `/api/routes?status=&role=` | 列表（筛选） | 按角色过滤字段 |
| GET | `/api/routes/:id` | 详情（含当前 version） | 字段级 |
| PUT | `/api/routes/:id` | 更新客户/状态 | 一手/对应旅行社 |
| POST | `/api/routes/:id/versions` | 保存并通知（新 version） | 一手/旅行社 |
| GET | `/api/routes/:id/versions` | 版本历史 | 一手/对应旅行社 |
| GET | `/api/routes/:id/versions/:vid` | 单版本 diff | 字段级 |

## 协作状态流转（服务端校验）
`待一手确认 ↔ 待旅行社修订`、`待一手确认 → 待报价`、`待报价 → 已报价 → …`
状态跃迁在 service 层强制校验，非法跃迁返回 409。
