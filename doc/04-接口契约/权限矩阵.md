# 接口契约：权限矩阵（字段级 + 物理隔绝）

## 角色 × 字段可见性
| 字段 | 一手 PandaKing | 境外旅行社 | 省地接社 |
|---|---|---|---|
| 客户档案 | ✓ | ✓(自身) | ✗ |
| 行程草案(旅行社) | ✓ | ✓ | ✗ |
| 成本①(省地接社成本) | ✓ | ✗ | ✓(自身) |
| 成本②(一手利润) | ✓ | ✗ | ✗ |
| 旅行社加价 | ✓ | ✓ | ✗ |
| 游客报价 | ✓ | ✓ | ✗ |
| 自身被询价成本价 | ✓ | ✗ | ✓(自身) |
| 他省地接社成本价 | ✓ | ✗ | ✗(隔离) |
| 知识库 | ✓(读写) | ✓(读) | ✗ |

## 物理隔绝（agency ↔ provincial 完全互不可见）
- **境外旅行社 与 省地接社 在价格与信息上完全物理隔绝**：互相不知道对方的存在、成员与数据展示。
- **路线列表**：一手见全部；境外旅行社仅见 `agency_id = 当前用户机构` 的路线；省地接社**无路线视图**（仅经「成本询价」交互，见下方）。
- **成本询价**：省地接社仅见 `agency_id = 当前用户机构` 的询价（他机构成本价 ✗ 隔离）。
- **成员列表**：一手见全部；机构成员仅见本机构成员（`WHERE agency_id = :currentAgencyId`）。
- 上述隔离在服务端强制执行（route 列表 / 详情 / 版本 / 反馈 / 成员 均按 agencyId 校验），前端隐藏仅为 UX。

## 机构内层级
- `admin`：可被授权邀请本机构 `staff`；`staff`：仅协作，不可邀请。
- 层级与机构归属存于 `User.level` / `User.agencyId` / `User.parentId`，由邀请继承。

## 实现要点
- 字段可见性在 **service 层** 用 `role` 投影，不靠前端隐藏（前端隐藏仅为 UX）。
- 省地接社 / 机构隔离：查询强制 `WHERE agency_id = :currentAgencyId`；境外旅行社路线列表同理。
- 所有写操作前校验操作人机构与路线归属一致。
- 报价字段剥离规则（`hideCostsForRole`）：一手全见；旅行社隐藏成本①/成本②（保留自身加价）；省地接社隐藏成本②/加价（保留自身成本①）；公开 H5 仅对客总价。

## 写操作权限（路线生命周期）
- **创建路线**：一手（必选境外旅行社）/ 境外旅行社（自动归属本机构）可创建；省地接社禁止。
- **删除路线**：**仅一手 PandaKing** 可删除。删除前整条路线（主记录 + 版本 + 共享链接 + 反馈 + 成本询价）快照写入 `RouteArchive` 备份历史库，再硬删；境外旅行社 / 省地接社无删除入口与权限（接口返回 403）。
- 状态流转（提交/确认/加价/报价/付款等）：一手与境外旅行社参与；省地接社不参与商业状态流转。

## 端点
| Method | Path | 说明 |
|---|---|---|
| GET | `/api/permissions/matrix` | 返回当前角色可见字段集（前端渲染用） |
| GET | `/api/me` | 当前用户角色与权限摘要 |
| DELETE | `/api/routes/:id` | 删除路线（仅一手；删除前归档快照到 `RouteArchive` 备份历史库） |
