## 四、上线前必须执行（迁移 + 部署）

代码已编译通过，但**数据库结构变了**，必须按顺序执行，否则运行时报列不存在：

1. 应用增量迁移（云托管 Postgres 无 `CREATE DATABASE` 权限，**禁用 `migrate dev`**）：
   ```bash
   cd backend
   pnpm migrate:deploy      # 应用 20260714230000_provincial_collaboration 等
   pnpm seed                # 重新 seed（演示省地接社需分配到种子路线）
   pnpm build
   ```
2. 重新部署前后端。
3. 验证：用一手账号将某路线 `assign-provincial` → 用省地接社账号登录应能看到该路线 → 生成协作 H5 → 省地接社打开可编辑行程。

> ⚠️ 迁移 SQL 为手写增量（`prisma/migrations/20260714230000_provincial_collaboration/migration.sql`），**未**包含云厂商内部探活表的 DROP（已剔除）。部署前勿用 `migrate diff` 重新生成覆盖。
